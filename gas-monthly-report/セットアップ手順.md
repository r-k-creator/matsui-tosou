# 月次サイト反応レポート GAS - セットアップ手順

このフォルダの `コード.gs` を、新規のGoogle Apps Scriptプロジェクトに設定するための手順です。
既存の「お問い合わせフォーム用GAS」とは完全に別プロジェクトとして作成してください。

## 1. 新規プロジェクトを作成
1. https://script.google.com/ を開く（nreo0525@gmail.comでログイン）
2. 「新しいプロジェクト」を作成
3. プロジェクト名を「松井塗装_月次サイト反応レポート」など分かりやすい名前に変更
4. デフォルトの `Code.gs` の中身を全て削除し、このフォルダの `コード.gs` の内容を貼り付けて保存

## 2. タイムゾーンの確認
1. 左メニューの歯車アイコン「プロジェクトの設定」を開く
2. 「関数のタイムゾーン」が `(GMT+09:00) 日本標準時 - Asia/Tokyo` になっているか確認（違う場合は変更）

## 3. Analytics Data API（高度なサービス）の有効化
1. 左メニューの「サービス」の横にある「+」をクリック
2. 一覧から「Analytics Data API」を探して選択し、「追加」をクリック
3. 追加後、左メニューの「サービス」に「AnalyticsData」が表示されていればOK

※もし実行時に「Analytics Data API が有効になっていません」というエラーが出た場合：
1. エラーメッセージ内のリンク（Google Cloud Consoleへのリンク）をクリック
2. 表示された画面で「有効にする」をクリック
3. 数分待ってから再実行

## 4. GA4側の閲覧権限の確認
1. https://analytics.google.com/ を開く
2. 対象プロパティ（プロパティID: 544263595、松井塗装のGA4プロパティ）を選択
3. 左下の「管理」→ プロパティ列の「アクセス管理」を開く
4. ユーザー一覧に `nreo0525@gmail.com` が含まれているか確認
   - 含まれていない場合は「+」から追加し、少なくとも「閲覧者」権限を付与

## 5. テスト実行
1. スクリプトエディタの関数選択で `testRun` を選ぶ
2. 「実行」をクリック
3. 初回は権限の承認画面が出るので、以下を許可する
   - スプレッドシートの読み取り
   - Googleアナリティクスデータの読み取り
   - メールの送信
4. `nreo0525@gmail.com` と `matsui.painter@gmail.com` の両方にテストメールが届くか確認
5. 届いた場合、実行ログ（表示 → 実行数）でエラーが出ていないかも確認

## 6. 本番用トリガーの設定
1. 左メニューの時計アイコン「トリガー」を開く
2. 右下「トリガーを追加」をクリック
3. 以下の内容で設定
   - 実行する関数：`main`
   - イベントのソース：時間主導型
   - 時間ベースのトリガーのタイプ：日タイマー
   - 時刻：午前8時～9時
4. 保存

このトリガーは毎日8時台に実行されますが、`main` 関数の中で「今日が15日 or 月末かどうか」を判定しているため、
実際にメールが送られるのは15日と月末の朝だけです。次回の実行予定はトリガー画面の「次回の実行」欄で確認できます。

## 7. 運用開始後の注意点
- コードを修正した場合は保存するだけで、次回のトリガー実行時から新しい内容が使われます（Webアプリのような「新しいデプロイ」は不要です）
- 前回集計との比較（前回比）は、スクリプトプロパティに自動保存される値をもとに計算されます。`testRun` はこの値を上書きしないので、テストのために本番の比較データが壊れることはありません
